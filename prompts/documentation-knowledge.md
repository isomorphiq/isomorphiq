# Documentation & Knowledge Sharing Prompts

## API Documentation Prompt
```
Create comprehensive API documentation for [SERVICE/ENDPOINT]:
- Endpoint descriptions and usage examples
- Request/response schemas
- Authentication requirements
- Error codes and handling
- Rate limiting information
- SDK/client library examples

Generate in OpenAPI format and provide interactive documentation.
```

## Code Documentation Prompt
```
Add comprehensive documentation to [COMPONENT/MODULE]:
- Function/class purpose and usage
- Parameter descriptions and types
- Return value specifications
- Exception/error conditions
- Usage examples and code samples
- Architecture decisions and design rationale

Use [DOCUMENTATION FORMAT] and ensure consistency.
```

## User Guide Creation Prompt
```
Create user documentation for [FEATURE/SYSTEM]:
- Getting started guide
- Feature walkthroughs
- Configuration options
- Troubleshooting guide
- Best practices and tips
- FAQ section

Target audience: [USER TYPE] with [EXPERIENCE LEVEL] experience.
```

## Technical Specification Prompt
```
Write technical specification for [FEATURE/SYSTEM]:
- Functional requirements
- Non-functional requirements
- System architecture
- Data models and schemas
- API specifications
- Security requirements
- Performance expectations

Include acceptance criteria and implementation guidelines.
```
